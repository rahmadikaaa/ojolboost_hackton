# OjolBoost: Multi-Agent Management System (MAMS) for Gig Workers

## 🚀 Project Overview
OjolBoost is an AI-orchestrated Multi-Agent Management System (MAMS) designed specifically for multi-platform gig economy workers (ride-hailing, food delivery, couriers) in Indonesia. It acts as a unified "Digital Copilot," addressing the fragmentation and cognitive overload faced by drivers who juggle multiple apps simultaneously. 

By leveraging Google's Gemini models natively integrated with Google Cloud (BigQuery, Cloud Run), OjolBoost automates financial tracking, schedule management, real-time demand heuristics, and environmental-based strategic pivots.

## 💡 The Problem
Gig workers juggle multiple applications (Gojek, Grab, Maxim, Lalamove) to maximize their daily income. This fragmentation forces them to mentally calculate net targets, predict high-demand areas blindly, and manually manage operational expenses—leading to cognitive fatigue, inefficient routing, and missed income opportunities.

## 🛠️ Our Solution
OjolBoost utilizes a strict Master-Subordinate AI architecture. A primary orchestrator agent ("Bang Jek") interacts naturally with the driver via voice/text and delegates specific tasks to specialized sub-agents. 

### Key Features:
1. **Target Reverse Calculator ("The Auditor"):** Drivers state their net target (e.g., "I need 250k net today"). The Auditor pulls historical cost data from BigQuery, calculates fuel/maintenance variables, and outputs exactly how many rides/food deliveries are needed to hit that target.
2. **Predictive Heatmaps ("Demand Analytics"):** Analyzing vast historical datasets in BigQuery to recommend the most profitable zones based on the current time and day.
3. **Context-Aware Pivot ("Environmental Agent"):** Pulls real-time weather data. If a storm hits, it advises the driver to pivot from "Ride" to "Food/Package" deliveries, maximizing surge pricing while minimizing passenger risk.
4. **Automated Reminders ("The Planner"):** Seamlessly schedules maintenance (e.g., oil changes) via Google Calendar integration (served over MCP).

## 🧠 Technical Approach & GCP Integration
We built OjolBoost with a non-deterministic frontend and a strictly deterministic backend to ensure safety and precision with financial data.

- **Primary Orchestrator (Vertex AI + Gemini 2.5 Flash):** Handles natural language understanding, intent extraction, and conversational synthesis. We employ strict Context Locking routing to prevent cross-talk between agents.
- **Data Engine (Google BigQuery):** Acts as the single source of truth for all historical demand data (`demand_history`) and financial tracking.
- **Serverless Automation (Google Cloud Run):** Hosts our backend operations and Model Context Protocol (MCP) servers, ensuring high availability, scalable inference, and zero-downtime deployments.
- **Guardrails (Deterministic Rules):** A crucial part of our architecture. Every SQL query generated by the AI (The Auditor/Demand Analytics) passes through a strict Python validation layer (checking dataset, table whitelist, and read/write permissions) before being executed against BigQuery, guaranteeing data integrity.
